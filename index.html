Comandos Básicos

SELECT

A instrução SELECT é usada para selecionar dados de um banco de dados.

Sintaxe

SELECT coluna1, coluna2, ...
FROM nome_tabela;

Exemplo

Retornar todas as colunas da tabela clientes:

SELECT * FROM Clientes;

WHERE

A cláusula WHERE é usada para filtrar registros.
É usado para extrair apenas os registros que atendem a uma condição especificada.

Sintaxe

SELECT coluna1, coluna2, ...
FROM nome_tabela
WHERE condição;

Exemplo

Selecionar todos os clientes do México:

SELECT * FROM Clientes
WHERE País = 'México';

ORDER BY

A palavra-chave ORDER BY é usada para classificar o conjunto de resultados em ordem crescente ou decrescente.

Sintaxe

SELECT coluna1, coluna2, ...
FROM nome_tabela
ORDER BY coluna1, coluna2, ... ASC|DESC;

Exemplo

Classifique os produtos por preço:

SELECT * FROM Produtos
ORDER BY Preço;

AND

A cláusula WHERE pode conter um ou vários operadores AND.
O operador AND é usado para filtrar registros com base em mais de uma condição.

Sintaxe

SELECT coluna1, coluna2, ...
FROM nome_tabela
WHERE condição1 AND condição2 AND condição3 ...;

Exemplo

​ Retornar todos os clientes da Espanha que comecem com a letra 'G':

SELECT *
FROM Clientes
WHERE País = 'Espanha' AND NomeCliente LIKE 'G%';

OR

A cláusula WHERE pode conter um ou mais operadores OR.
O operador OR é usado para filtrar registros com base em mais de uma condição

Sintaxe

SELECT coluna1, coluna2, ...
FROM nome_tabela
WHERE condição1 OR condição2 OR condição3 ...;

Exemplo

​ Retornar todos os clientes da Alemanha, mas também os da Espanha:

SELECT *
FROM Clientes
WHERE País = 'Espanha' OR País = 'Alemanha';

NOT

O operador NOT é usado em combinação com outros operadores para fornecer o resultado oposto, também chamado de resultado negativo.

Sintaxe

SELECT coluna1, coluna2, ...
FROM nome_tabela
WHERE NOT condição;

Exemplo

​ Devolver todos os clientes que NÃO são da Espanha:

SELECT * FROM Clientes
WHERE NOT País = 'Espanha';

INSERT

A instrução INSERT INTO é usada para inserir novos registros em uma tabela.

Sintaxe

1. Especifique os nomes das colunas e os valores a serem inseridos:

INSERT INTO nome_tabela (coluna1, coluna2, coluna3, ...)
VALUES (valor1, valor2, valor3, ...);

2. Se estiver adicionando valores para todas as colunas da tabela, não será necessário especificar os nomes das colunas na consulta SQL. No entanto, certifique-se de que a ordem dos valores esteja na mesma ordem das colunas da tabela. Aqui, a sintaxe INSERT INTO seria a seguinte:

INSERT INTO nome_tabela
VALUES (valor1, valor2, valor3, ...);

Exemplo

​ Insere um novo registro na tabela "Clientes":

INSERT INTO Clientes (nomeCliente, contactoNome, Endereço, Cidade, CodigoPostal, País)
VALUES ('Cardinal', 'Tom B. Erichsen', 'Skagen 21', 'Stavanger', '4006', 'Norway');

UPDATE

A instrução UPDATE é usada para modificar os registros existentes em uma tabela.

Sintaxe

UPDATE nome_tabela
SET coluna1 = valor1, coluna2 = valor2, ...
WHERE condição;

Exemplo

​ A instrução SQL a seguir atualiza o primeiro cliente (ClienteID = 1) com uma nova pessoa de contato e uma nova cidade.

UPDATE Clientes
SET contactoNome = 'Alfredo Silva', Cidade = 'Frankfurt'
WHERE ClienteID = 1;

DELETE

A instrução DELETE é usada para excluir registros existentes em uma tabela.

Sintaxe

DELETE FROM nome_tabela WHERE condição;

Exemplo

​ A seguinte instrução SQL exclui o cliente "António das Rosas" da tabela "Clientes":

DELETE FROM Clientes WHERE nomeCliente ='António das Rosas';

MIN e MAX

A função MIN() retorna o menor valor da coluna selecionada.
A função MAX() retorna o maior valor da coluna selecionada.

Sintaxe

SELECT MIN/MAX(nome_coluna)
FROM nome_tabela
WHERE condição;

Exemplo

​ Encontrar o preço mais baixo:

SELECT MIN(Preço)
FROM Produtos;

​ Encontrar o preço mais alto:

SELECT MAX(Preço)
FROM Produtos;

COUNT

A função COUNT() retorna o número de linhas que corresponde a um critério especificado.

Sintaxe

SELECT COUNT(nome_coluna)
FROM nome_tabela
WHERE condição;

Exemplo

​ Encontrar o número total de produtos na tabela Produtos:

SELECT COUNT(*)
FROM Produtos;

SUM

A função SUM() retorna a soma total de uma coluna numérica.

Sintaxe

SELECT SUM(nome_coluna)
FROM nome_tabela
WHERE condição;

Exemplo

​ Retornar a soma de todos os campos Quantidade na tabela DetalhesEncomenda:

SELECT SUM(Quantidade)
FROM DetalhesEncomenda;

AVG

A função AVG() retorna o valor médio de uma coluna numérica.

Sintaxe

SELECT AVG(nome_coluna)
FROM nome_tabela
WHERE condição;

Exemplo

​ Encontrar o preço médio de todos os produtos:

SELECT AVG(Preço)
FROM Produtos;

LIKE

O operador LIKE é usado em uma cláusula WHERE para procurar um padrão especificado em uma coluna.

Existem dois curingas frequentemente usados em conjunto com o operador LIKE:

- O sinal de porcentagem % representa zero, um ou vários caracteres
- O sinal de sublinhado _ representa um único caractere

Sintaxe

SELECT coluna1, coluna2, ...
FROM nome_tabela
WHERE colunaN LIKE padrão;

Exemplo

​ Selecione todos os clientes que começam com a letra "a":

SELECT * FROM Clientes
WHERE NomeCLiente LIKE 'A%';

IN

O operador IN permite especificar vários valores em uma cláusula WHERE.
O operador IN é uma abreviatura para múltiplas condições OR.

Sintaxe

SELECT nome_coluna(s)
FROM nome_tabela
WHERE nome_coluna IN (valor1, valor2, ...);

Exemplo

​ Retornar todos os clientes da 'Alemanha', 'França' ou 'Reino Unido':

SELECT * FROM Clientes
WHERE País IN ('Alemanha', 'França', 'Reino Unido');

BETWEEN

O operador BETWEEN seleciona valores dentro de um determinado intervalo. Os valores podem ser números, texto ou datas.
O operador BETWEEN é inclusivo: os valores inicial e final estão incluídos.

Sintaxe

SELECT nome_coluna(s)
FROM nome:tabela
WHERE nome_coluna BETWEEN valor1 AND valor2;

Exemplo

​ Seleciona todos os produtos com preço entre 10 e 20:

SELECT * FROM Produtos
WHERE Preço BETWEEN 10 AND 20;

Aliases

Aliases SQL são usados para fornecer um nome temporário a uma tabela ou coluna em uma tabela.
Aliases são frequentemente usados para tornar os nomes das colunas mais legíveis.
Um alias existe apenas durante essa consulta.
Um alias é criado com a palavra-chave AS.

Sintaxe

Quando é usado numa coluna:

SELECT nome_coluna AS nome_alias
FROM nome_tabela;

Quando é usado numa tabela:

SELECT nome_tabela(s)
FROM nome_tabela AS nome_alias;

Exemplo

​ Seleciona todos os produtos com preço entre 10 e 20:

SELECT ClienteID AS ID
FROM Clientes;

INNER JOIN

A palavra-chave INNER JOIN seleciona registros que possuem valores correspondentes em ambas as tabelas.

Sintaxe

SELECT nome_coluna(s)
FROM tabela1
INNER JOIN tabela2
ON tabela1.nome_coluna = tabela2.nome_coluna;

Exemplo

​ Juntar produtos e categorias com a palavra-chave INNER JOIN:

SELECT ProdutoID, ProdutoNome, CategoriaNome
FROM Produtos
INNER JOIN Categorias ON Produtos.CategoriaID = Categorias.CategoriaID;

LEFT JOIN

A palavra-chave LEFT JOIN retorna todos os registros da tabela esquerda (tabela1) e os registros correspondentes da tabela direita (tabela2). O resultado é 0 registros do lado direito, se não houver correspondência.

Sintaxe

SELECT nome_coluna(s)
FROM tabela1
LEFT JOIN tabela2
ON tabela1.nome_coluna = tabela2.nome_coluna;

Exemplo

​ A instrução SQL a seguir selecionará todos os clientes e quaisquer pedidos que eles possam ter:

SELECT Clientes.NomeCliente, Encomendas.EncomendaID
FROM Clientes
LEFT JOIN Encomendas ON Clientes.CienteID = Encomendas.ClienteID;
ORDER BY Cientes.NomeCliente;

RIGHT JOIN

A palavra-chave RIGHT JOIN retorna todos os registros da tabela direita (tabela2) e os registros correspondentes da tabela esquerda (tabela1). O resultado são 0 registros do lado esquerdo, se não houver correspondência.

Sintaxe

SELECT nome_coluna(s)
FROM tabela1
RIGHT JOIN tabela2
ON tabela1.nome_coluna = tabela2.nome_coluna;

Exemplo

​ A instrução SQL a seguir retornará todos os funcionários e quaisquer pedidos que eles possam ter feito:

SELECT Encomendas.EncomendaID, Empregado.UltimoNome, Empregado.PrimeiroNome
FROM Encomendas
RIGHT JOIN Empregados ON Encomendas.EmpregadoID = Empregados.EmpregadoID
ORDER BY Encomendas.EncomendaID;

FULL OUTER JOIN

A palavra-chave FULL OUTER JOIN retorna todos os registros quando há uma correspondência nos registros da tabela esquerda (tabela1) ou direita (tabela2).
Dica: FULL OUTER JOIN e FULL JOIN são iguais.

Sintaxe

SELECT nome_coluna(s)
FROM tabela1
FULL OUTER JOIN tabela2
ON tabela1.nome_coluna = tabela2.nome_coluna; WHERE condição;

Exemplo

​ A seguinte instrução SQL seleciona todos os clientes e todos os pedidos:

SELECT Clientes.NomeCliente, Encomendas.EncomendaID
FROM Clientes
FULL OUTER JOIN Encomendas ON Clientes.ClienteID = Encomenda.ClienteID
ORDER BY Clientes.NomeCliente;

CROSS JOIN

A palavra-chave CROSS JOIN retorna todos os registros de ambas as tabelas (tabela1 e tabela2).

Sintaxe

SELECT nome_coluna(s)
FROM tabela1
CROSS JOIN tabela2;

Exemplo

​ A seguinte instrução SQL seleciona todos os clientes e todos os pedidos:

SELECT Clientes.NomeCliente, Encomendas.EncomendaID FROM Clientes CROSS JOIN Encomendas;

CREATE TABLE

A instrução CREATE TABLE é usada para criar uma nova tabela em uma base de dados.

Sintaxe

CREATE TABLE nome_tabela (
coluna1 tipodado,
coluna2 tipodado,
coluna3 tipodado,
....
);

Exemplo

​ O exemplo a seguir cria uma tabela chamada "Pessoas" que contém cinco colunas: PessoaID, UltimoNome, PrimeiroNome, Endereco e Cidade. Mostra também como definir a chave primária e chaves estrangeiras.

CREATE TABLE Persons (
PessoaID int,
UltimoNome varchar(255),
PrimeiroNome varchar(255),
Endereco varchar(255),
CidadeID varchar(255)
PRIMARY KEY (PessoaID)
FOREIGN KEY (CidadeID) REFERENCES Cidades(CidadesID)
);

Índices

CREATE INDEX

A instrução CREATE INDEX é usada para criar índices em tabelas.
Os índices são usados para recuperar dados da base de dados mais rapidamente do que de outra forma. Os usuários não podem ver os índices, eles apenas servem para agilizar pesquisas/consultas.

Sintaxe

1. Cria um índice numa tabela. Valores duplicados são permitidos:

CREATE INDEX nome_indice ON nome_tabela (coluna1, coluna2, ...);

2. Cria um índice único numa tabela. Valores duplicados não são permitidos:

CREATE UNIQUE INDEX nome_indice ON nome_tabela (coluna1, coluna2, ...);

Exemplo

​ A instrução SQL abaixo cria um índice denominado "indice_ultimoNome" na coluna "UltimoNome" da tabela "Pessoas":

CREATE INDEX indice_ultimoNome ON Pessoas (UltimoNome);

Views

CREATE VIEW

Em SQL, uma view é uma tabela virtual baseada no conjunto de resultados de uma instrução SQL.
Uma view contém linhas e colunas, assim como uma tabela real. Os campos em uma view são campos de uma ou mais tabelas reais do banco de dados.
Você pode adicionar instruções e funções SQL a uma view e apresentar os dados como se viessem de uma única tabela.
Uma view é criada com a instrução CREATE VIEW.

Sintaxe

CREATE VIEW nome_view AS
SELECT coluna1, coluna2, ...
FROM nome_tabela
WHERE condição;

Exemplo

​ O SQL a seguir cria uma view que mostra todos os clientes do Brasil:

CREATE VIEW [Clientes do Brasil] AS SELECT NomeCliente, NomeContacot FROM Clientes WHERE País = 'Brasil';

Tipos de Dados MySQL

Em MySQL existem três principais tipos de dados: string, numérico e data e hora.

Alguns exemplos de para cada um deles são:

String: CHAR, VARCHAR, TEXT, LONGTEXT
Numérico: INT, FLOAT, DOUBLE, BOOL
Data e Hora DATE, DATETIME, TIME, YEAR
